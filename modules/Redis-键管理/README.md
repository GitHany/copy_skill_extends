# Redis 键管理

Redis 键管理常用命令参考。

## KEYS 查找键

**命令**: `KEYS %{模式}%`

**参数说明**:
| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| 模式 | string | 是 | 键名匹配模式（支持 * ? [] 通配符）【常用】 | user:* |

**扩展示例**:

| 场景 | 命令 | 说明 |
|------|------|------|
| 查找所有键 | `redis-cli KEYS '*'` | 模式=* 匹配所有键 |
| 查找用户键 | `redis-cli KEYS 'user:*'` | 模式=user:*【常用】 |
| 查找带前缀的键 | `redis-cli KEYS 'cache:*'` | 模式=cache:* |

| 场景 | 命令 | 说明 |

| 本数据库复制 | `redis-cli COPY user:profile user:backup` | 复制到同一数据库的不同键名 |

## SCAN 扫描键

**命令**: `SCAN %{游标}% %{匹配模式}%`

**参数说明**:
| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| 游标 | number | 是 | 游标位置（首次使用0，之后使用返回的游标） | 0 |
| 匹配模式 | string | 否 | 键名匹配模式（支持 * ? [] 通配符） | user:* |

**扩展示例**:

| 场景 | 命令 | 说明 |
|------|------|------|
| 扫描所有键 | `redis-cli SCAN 0` | 游标=0，开始扫描 |
| 扫描匹配模式的键 | `redis-cli SCAN 0 MATCH user:*` | 只返回匹配user:*的键 |
| 带COUNT限制 | `redis-cli SCAN 0 COUNT 100` | 每次最多返回100个键 |
